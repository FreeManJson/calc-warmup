# 冒険モード v1 / stage1 まとめ

## 今回の切り方

### 第1弾の目的
- 通常モードを壊さずに、冒険モードの土台を横に追加する
- フレーバー外出しの最小単位として `theme` 定義を先に導入する
- 今後の拡張で壊れやすい「進行状況保存」「結果保存」「敵進行」「報酬反映」を先に固定する

### 第1弾で入れた最小仕様
- TOP から通常モード / 冒険モードを分岐
- 冒険モード専用ページ追加
- 60秒固定のダンジョン挑戦
- 4人パーティの総攻撃力制
- 敵順固定: ザコA → ザコB → ザコC → ボス → 条件付きシークレット
- 正答 = 総攻撃力ダメージ / 誤答 = 0ダメージ
- ボスを30秒以内に倒した場合のみシークレット出現
- 素材獲得
- 素材5個ごとの自動錬成
- 4人分そろったら宝解放
- 冒険結果画面追加
- ユーザー別に冒険進行状況を localStorage 保存
- ダンジョン名 / 敵名 / 素材名 / 宝名 / 武器名 / 職業名を theme ファイルへ分離

## 今回の構造方針

### 1. ロジックとフレーバーの分離
- `src/themes/defaultAdventureTheme.ts`
  - 名称、敵名、武器名、素材名、宝名、アイコン名、背景名
- `src/utils/adventureUtils.ts`
  - 60秒、30秒、HP係数、素材数、攻撃力上昇量などのゲームロジック

この分離により、将来は theme を差し替えても、戦闘ロジックや成長ロジックは基本維持できます。

### 2. 保存単位
- 通常モード設定: 既存どおりユーザー別保存
- 冒険進行: ユーザー別保存
- 冒険最新結果: 別キーで保存

### 3. 今後の拡張ポイント
- continue / 終了 選択
- コレクション画面
- トロフィー画面
- 武器・宝の一覧画面
- 複数テーマ切替
- 画像アセットの実装
- 通常モード設定UIの選択式化
